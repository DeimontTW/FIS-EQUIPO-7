Criterios de Aceptación

| Requisitos Funcionales | Criterios de Aceptación |
| :---- | :---- |
| El mapa debe representar la infraestructura de la facultad de matemáticas en un esquema detallado en 2D y un entorno virtual Street View | El sistema muestra correctamente el mapa 2D y el entorno virtual tipo Street View, de manera clara y completa de los edificios y áreas de la facultad |
| Mostrar la ubicación del usuario y rastrearla usando el GPS de su dispositivo | El sistema detecta y actualiza la ubicación del usuario en tiempo real dentro del mapa |
| Alternar entre la vista 2D y el entorno Street View en una transición cuando el usuario así lo desee | El usuario puede cambiar entre el mapa 2D y el Street View, cuando desee, con un solo clic, sin interrupciones ni recarga completa del sitio web |
| El entorno Street View deberá contar con botones de navegación en forma de flechas para permitir la navegación del usuario (arriba, abajo, izquierda, derecha) | El entorno Street View permite desplazarse en las cuatro direcciones mediante botones en forma de flechas visibles y funcionales, manteniendo la orientación correcta |
| Mostrará por medio de leyendas y símbolos, puntos de interés de la facultad, representados por colores | Todas las leyendas y puntos de interés están identificados mediante colores y símbolos visibles |
| Al elegir un punto de interés, aparecerá un apartado con su información general, horario de atención y contacto. | Al seleccionar un punto de interés, el sistema muestra correctamente su nombre, descripción, horario y medio de contacto en un panel informativo |
| El mapa enviará notificaciones push cuando ya se encuentre abierta la cafetería, sala de cómputo, biblioteca, entre otros puntos de interés | El sistema genera y envía notificaciones push de manera automática en el dispositivo del usuario, cada vez que ocurre un evento de apertura por cada punto de interés |
| Incluirá una herramienta de búsqueda para puntos de interés y planificar una ruta con punto de inicio y destino | El buscador permite al usuario encontrar un punto de interés por nombre, mostrar su ubicación y ruta con punto de inicio y destino |
| Opción de traducción de idioma (Español-Inglés) | El sistema permite visualizar todo el texto del sitio en español e inglés, mediante una opción de selección de idioma, sin errores de traducción |
| Apartado de configuración para activar o desactivar los tooltips y animaciones del mapa | El sistema cuenta con una sección de configuración, donde el usuario puede decidir si activar o desactivar los tooltips y animaciones del mapa, aplicando los cambios de forma inmediata. |

| Requisitos No Funcionales | Criterios de Aceptación |
| :---- | :---- |
| El sistema debe restringir la visualización y el rastreo de la ubicación del usuario únicamente cuando este se encuentre físicamente dentro del perímetro geográfico de la Facultad de Matemáticas, utilizando geofencing para validar la posición. | Si un usuario se encuentra fuera del perímetro de la Facultad de Matemáticas cuando solicite visualizar su ubicación en el mapa, el sistema debe de bloquear el acceso y mostrar un mensaje que diga: “Esta funcionalidad solo está disponible dentro del campus”. |
| El sistema debe garantizar una disponibilidad del 99.999% mensual| Dentro de un rango de 30 días consecutivos y se monitoreé el tiempo de operación del sistema, este deberá de registrar un tiempo total de inactividad no planificada no debe exceder 4.32 minutos por mes |
| La navegación entre secciones del sistema no debe superar una latencia de 3 segundos por interacción en condiciones de red estándar (al menos 5 Mbps de bajada). | Si el usuario cuenta con una conexión a internet al momento de realizar cualquier acción de navegación en el mapa, este debe responder en tiempo menor a 3 segundos. |
| El sitio debe ser completamente funcional y visualmente coherente en el navegador modernos más utilizado (Chrome), adaptándose mediante diseño responsivo. | Si el navegador Chrome se usan para acceder al sistema, el sistema no deberá perder ninguna funcionalidad y sin ocasionar errores en los siguientes dispositivos : Dispositivo movil, Tablet y Desktop. |
| El sistema debe implementar cifrado HTTPS en todas las comunicaciones para proteger los datos del usuario. | Cualquier petición HTTP, al momento de transmitir los datos del usuario, el sistema deberá de redirigir de manera automática a HTTPS y presentar un certificado de Secure Sockets Layer (SSL) válido. |
| El sistema debe estar documentado utilizando estándares de desarrollo web para facilitar futuras actualizaciones o corrección de errores. | Si el repositorio donde se encuentra el código fuente se llegara a revisar, este debe de contar con la documentación en un formato adecuado para su revisión. |
| Las imágenes mostradas en la vista tipo Street View deben aplicar un desenfoque automático (blur) sobre los rostros humanos detectados, utilizando técnicas de anonimización visual antes de su publicación, para proteger la identidad y privacidad de las personas. | Si cualquier imagen capturada para el formato Street View al momento de procesarlas, el sistema debe de aplicar un algoritmo de detección facial que desenfoque el 100% de los rostros con blur. |
| El sistema debe soportar un mínimo de 300 usuarios activos concurrentes accediendo simultáneamente a las funcionalidades principales (navegación del mapa, visualización en Street View y consulta de información de espacios) sin degradación perceptible en el rendimiento. | Si 300 usuarios usan el sistema de manera simultánea, al momento de ejecutar las funciones de navegación del mapa, Street View, la consulta de los espacios y de los puntos de interés, el sistema deberá de mantenerse con un tiempo de respuesta menor a 3 segundos. |

| Historias de Usuario | Criterios de Aceptación |
| :---- | :---- |
| Como aspirante quiero un mapa de la facultad para conocer sus instalaciones antes de ingresar a la universidad | El sistema muestra un mapa completo de la facultad, con las principales áreas, edificios y servicios visibles, permitiendo la libre exploración de la facultad |
| Como aspirante quiero poder navegar por el mapa para conocer todas las áreas de la universidad | El usuario puede desplazarse libremente por el mapa en vista 2D o Street View sin errores de carga |
| Como aspirante quiero acceder desde mi dispositivo para poder visualizarlo en línea sin necesidad de ir presencialmente | El sistema es accesible desde navegadores móviles y de escritorio, sin requerir instalaciones |
| Como aspirante quiero visualizar las áreas principales y su información general para conocer los servicios que ofrece la universidad | El sistema muestra correctamente las áreas principales y al seleccionarlas, despliega información clara sobre los servicios de la universidad |
| Como estudiante quiero buscar mi salón en el mapa para ubicarlo con facilidad | El estudiante puede ingresar el nombre del salón en el buscador, y el sistema muestra su ubicación exacta |
| Como estudiante quiero conocer la ruta más fácil para llegar al lugar donde necesito acudir a tiempo | El sistema genera una ruta visible y viable, entre el punto de inicio y el punto de destino, mostrando el recorrido más corto dentro de la facultad |
| Como estudiante quiero encontrar y conocer los puntos de interés para acudir a ellos cuando lo necesite | El mapa muestra los puntos de interés y permite visualizar información de cada uno al seleccionarlos |
| Como estudiante quiero recibir notificaciones cuando una dependencia esté abierta para poder acudir a ella | El sistema envía notificaciones push al dispositivo, que indican que dependencias están abiertas |
| Como estudiante quiero enterarme de avisos relevantes para estar al tanto de lo que sucede en la facultad | El sistema muestra o envía avisos importantes en tiempo real |
| Como visitante quiero conocer mi ubicación dentro de la facultad para poder ubicarme dentro de esta | El usuario externo puede ver su ubicación en el mapa mediante el GPS de su dispositivo, mientras se encuentra dentro de la facultad |
| Como visitante quiero usar un buscador para encontrar el lugar al que necesito acudir | El visitante puede buscar un lugar y el sistema muestra su ubicación con un marcador |
| Como persona externa quiero conocer la ruta más fácil para llegar al lugar donde necesito acudir a tiempo | El sistema genera la ruta visual más fácil y rápida,desde el punto actual del usuario al punto de destino seleccionado |
