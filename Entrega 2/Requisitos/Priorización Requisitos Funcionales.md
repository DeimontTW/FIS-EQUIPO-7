**Prioridad media**  **Prioridad baja**

* **El mapa debe representar la infraestructura de la facultad de matemáticas en un esquema detallado en 2D y un entorno virtual Street view** : Este requisito es el más indispensable pues forma el eje central del sistema

* **Mostrar la ubicación del usuario y rastrearla usando el GPS de su dispositivo** :  Importante para que el usuario tenga un punto de referencia y así poder guiarse, mejorando su experiencia por la facultad

* **El entorno Street View deberá contar con botones de navegación en forma de flechas para permitir la navegación del usuario (arriba, abajo, izquierda, derecha)** : Esencial para que el usuario pueda recorrer el entorno y conocerlo

* **Mostrará por medio de leyendas y símbolos puntos de interés de la facultad, representados por colores**: Importante para que el usuario pueda conocer los lugares de la facultad e identificarlos

* **Alternar entre la vista en 2D y el entonrno Street View en una transición cuando el usuario así lo desee** : Necesaria para que el usuario se familiarize con el mapa y ofrecer una buena experiencia de uso

* **Incluirá una heramienta de búsqueda para puntos de interés y planificar una ruta con punto de inicio y destino** : Esencial para evitar la desorientación y facilitar la búsqueda y la ruta a seguir que el usuario necesite

* **El mapa enviará notificaciones push cuando ya se encuentre abierta la cafetería, sala de cómputo, biblioteca, entre otros puntos de interés** : Requerimiento para mantener informado al usuario sobre eventos que ocurran en determinado momento notificándolos de manera atractiva

* **Apartado de configuración para activar o desactivar tooltips y animaciones del mapa** : Para simplificar el uso del mapa si el usuario así lo desea

* **Opción de traducción de idioma (Español-Inglés)** : Permite a usuarios que hablen el idioma inglés usar el mapa
